# AGENTS.md

В проекте есть единая база знаний для ИИ-агентов: [`_agent_knowledge`](./_agent_knowledge). Она является общим входом в проектный контекст и не привязана к одному конкретному инструменту.

Перед любой предметной работой агент обязан:
1. Открыть [`_agent_knowledge/11_KNOWLEDGE_ROUTER.md`](./_agent_knowledge/11_KNOWLEDGE_ROUTER.md).
2. Открыть [`_agent_knowledge/03_AGENT_RULES.md`](./_agent_knowledge/03_AGENT_RULES.md).
3. Через маршрутизатор перейти в профильные файлы по текущей задаче.

Базовый контекст:
- Проект: «Список рядом».
- Основная среда разработки: Claude Code.
- Вспомогательные агенты и инструменты могут включать Codex, Cursor и другие решения, но все они должны использовать одну и ту же knowledge-базу.
- Основная IDE: Android Studio.

Правило завершения каждой итерации:
- После каждого изменения проекта агент обязан проверить, не требуется ли актуализация файлов в `_agent_knowledge`.
- Если требуется, нужные knowledge-файлы обновляются сразу в той же итерации.
- Протокол проверки и обновления описан в [`_agent_knowledge/12_UPDATE_PROTOCOL.md`](./_agent_knowledge/12_UPDATE_PROTOCOL.md).
- Перед закрытием задачи агент должен вернуться к обязательному минимуму из `12_UPDATE_PROTOCOL.md` и проверить, не устарели ли `01_CURRENT_STATE.md`, `08_DECISION_LOG.md`, `09_CURRENT_FOCUS.md`, `13_CHANGELOG.md` и профильные файлы по теме.

`AGENTS.md` не дублирует всю базу знаний. За контекстом, ограничениями и текущим фокусом агент должен идти в профильные файлы внутри `_agent_knowledge`.
