# 02_USER_RULES

**Назначение:** хранить пользовательские правила и ограничения, которые агент обязан соблюдать в любой задаче.  
**Когда читать:** перед выдачей объяснений, инструкций, команд или предложений по следующему шагу.  
**Когда обновлять:** если пользователь добавляет новые обязательные правила работы, меняет формат взаимодействия или уточняет ограничения по задачам и командам.

## Роль пользователя

- Пользователь не является программистом.
- Нельзя требовать от пользователя самостоятельного проектирования архитектуры.
- Нельзя перекладывать на пользователя написание кода с нуля.
- Нельзя подталкивать пользователя к техническим решениям без понятного объяснения их смысла и последствий.

## Как давать команды

Если агент даёт пользователю команды для PowerShell, он обязан:
- выдавать их в готовом порядке для последовательного копирования и вставки;
- в каждом новом блоке команд первой строкой ставить `cd /d D:\projects\spisok-ryadom`;
- не перемешивать команды и не давать неудобный порядок действий;
- не просить пользователя самостоятельно перестраивать команды под ситуацию, если это можно сделать заранее.

## Как вести работу

- Работа идёт строго последовательно.
- Нельзя забегать вперёд до завершения текущего шага.
- Один запрос агенту — одно содержательное изменение.
- Нельзя смешивать несколько крупных несвязанных задач в одном запросе.
- Нужно предлагать узкие, предметные и проверяемые действия.

## Как оценивать изменения

- Нельзя ломать уже согласованную концепцию без явной фиксации нового решения.
- Любое новое изменение должно оцениваться на предмет влияния на уже согласованную модель приложения.
- Если затрагиваются архитектура, сущности, интерфейс, ограничения версии или сценарии, это нужно явно учитывать до внесения правки.
