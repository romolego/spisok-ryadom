# 03_AGENT_RULES

**Назначение:** зафиксировать обязательное поведение ИИ-агента внутри проекта и единый порядок работы с базой знаний.  
**Когда читать:** сразу после `AGENTS.md`, до начала любой предметной работы.  
**Когда обновлять:** если меняется агентский workflow, обязательный порядок чтения, правила актуализации knowledge-базы или распределение ролей между инструментами.

## Обязательный старт каждой новой задачи

Перед началом новой задачи агент обязан:
1. Прочитать `AGENTS.md`.
2. Прочитать `11_KNOWLEDGE_ROUTER.md`.
3. Прочитать этот файл.
4. Открыть профильные knowledge-файлы по текущей задаче.

Агент не должен начинать предметную работу, игнорируя проектную базу знаний.

## Роль агента в этом проекте

Агент отвечает за:
- практическую реализацию изменений;
- аккуратное внесение правок без расползания по проекту;
- поддержание проектной структуры;
- поддержание актуальности базы знаний после каждой завершённой итерации.

Пользователь не должен оставаться один на один с архитектурой, кодом и техническими решениями без понятного объяснения.

## Общие правила поведения

- Работать по принципу «одно содержательное изменение, одна итерация».
- Не объединять несколько крупных несвязанных задач в одну правку.
- Не подменять согласованную концепцию собственными догадками.
- Использовать knowledge-базу одинаково вне зависимости от того, работает ли агент в Claude Code, Codex, Cursor или другом инструменте.
- Не создавать скрытых правил, которые понятны только одному конкретному агенту.

## Инструменты проекта

- Основная среда разработки в проекте — Claude Code.
- Codex и Cursor остаются подтверждёнными вспомогательными инструментами проекта.
- База знаний должна оставаться пригодной и для других агентов и будущих инструментов.
- Любой агент должен ориентироваться на одну и ту же knowledge-инфраструктуру, а не на привычки конкретного инструмента.

## Обязательная проверка после каждого изменения

После каждого изменения проекта агент обязан проверить, какие файлы базы знаний устарели.

Перед завершением задачи агент обязан вернуться к `12_UPDATE_PROTOCOL.md`, проверить обязательный минимум knowledge-файлов и при необходимости открыть `14_SOURCE_CONTEXT.md`, если возникло сомнение в источнике истины.

Если изменения затрагивают:
- архитектуру;
- сущности и связи;
- функционал;
- интерфейсные правила;
- ограничения первой версии;
- текущее состояние проекта;
- проектные решения;
- текущий фокус;

то агент обязан актуализировать соответствующие файлы в `_agent_knowledge` сразу в той же итерации.

Подробный порядок действий описан в `12_UPDATE_PROTOCOL.md`.
