# 06_FUNCTIONAL_REQUIREMENTS

**Назначение:** хранить функциональные требования проекта без смешивания с технической реализацией.  
**Когда читать:** при работе с пользовательскими сценариями, логикой экранов, поведением сущностей и определением границ первой версии.  
**Когда обновлять:** если меняются функции первой версии, пользовательские сценарии, состав представлений данных или границы отложенных возможностей.

## Какие функции обязательны

Для целевой первой версии должны поддерживаться:
- ведение собственной базы товаров;
- ведение базы магазинов и отделов магазинов;
- добавление товаров из базы в один общий текущий список;
- создание новой карточки товара из разных точек интерфейса как допустимый вторичный сценарий;
- назначение магазина и, при необходимости, отдела;
- хранение количества и единицы измерения;
- отметка «куплено / не куплено»;
- срочность у записи текущего списка;
- очистка купленных записей без удаления карточек товаров из базы;
- быстрый перенос позиции между допустимыми магазинами;
- простой локальный поиск по базе товаров;
- локальное хранение данных.

## Какие представления данных нужны

Приложению нужны представления:
- по магазинам;
- по общему справочнику товаров;
- по пользовательским группам товаров;
- по получателям;
- по общему текущему списку без привязки к магазинам.

Главным рабочим представлением остаётся представление по магазинам.

В этой knowledge-базе формулировка «пользовательские группы» относится к пользовательским группам товаров, а не к отдельным пользовательским аккаунтам или командам.

## Какие действия пользователя считаются базовыми

Базовые действия пользователя:
- создать и отредактировать карточку товара;
- создать и отредактировать магазин;
- задать отделы магазина;
- привязать товар к одному или нескольким магазинам;
- выбрать существующий товар из базы;
- добавить товар в текущий список;
- изменить количество;
- назначить или сменить магазин;
- назначить или сменить отдел;
- отметить позицию как купленную и вернуть её обратно при ошибке;
- удалить запись из текущего списка без удаления карточки товара.

## Что входит в первую версию

В общую согласованную первую версию входят:
- один общий текущий список;
- карточки товаров и магазинов;
- отделы магазинов;
- пользовательские группы товаров и получатели;
- поиск похожих товаров;
- подсказки по названию, которые рекомендуют уже существующие карточки, но не блокируют создание новой;
- единицы измерения с подсказками вместо жёсткого закрытого справочника;
- приоритетный магазин как магазин по умолчанию при добавлении из общего контекста;
- фото товара, заметки и базовая онлайн/оффлайн логика;
- история состояния списка и изменений как часть общей концепции v1.

Это целевой объём первой версии в целом. Он не означает, что все перечисленные возможности должны появиться уже в ближайшей каркасной итерации.

## Что отложено на потом

На более поздние итерации можно выносить:
- расширенную работу со справочником «Не покупать»;
- более сложную аналитику истории;
- шаблоны повторных покупок;
- продвинутую работу с ценами;
- импорт, экспорт и массовое редактирование;
- дополнительные механики вокруг онлайн-заказов.

## Какие сценарии главные

Главные сценарии проекта:
- собрать текущую покупку из уже существующей базы товаров;
- видеть покупки по магазинам в пользовательском порядке;
- быстро пройтись по магазину, отмечая купленное;
- не потерять отложенные и несрочные позиции;
- быстро перенести не найденный товар в другой допустимый магазин.

## Какие сценарии вторичны

Вторичными считаются:
- создание новой карточки товара вместо выбора из базы;
- расширенные сценарии истории;
- углублённая работа с ценами;
- расширенный сценарий «Не покупать».
