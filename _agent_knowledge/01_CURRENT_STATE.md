## Что уже реализовано в коде

### Быстрое редактирование записи текущего списка
- Основное взаимодействие с записью списка — это работа с текущей покупкой, а не с карточкой товара.
- Короткий тап по любой основной зоне карточки записи (название товара, детали, заметка, бейдж количества) открывает диалог быстрого редактирования записи. Чекбокс «куплено» и иконка срочности обрабатываются отдельно и не конфликтуют с общим тапом.
- Справа от каждой записи расположен бейдж количества (например, «2 л») как визуальный индикатор, но зона клика охватывает всю карточку.
- Диалог EditEntryDialog позволяет изменить количество (единица измерения показана как контекст, без редактирования), назначить магазин (с приоритетом связанных магазинов) и добавить заметку.
- Количество и единица измерения в записи текущего списка (ShoppingListEntryEntity.quantity/unit) независимы от значений по умолчанию в карточке товара (ProductEntity.defaultQuantity/defaultUnit).
- Значения по умолчанию из карточки товара используются только как стартовая подстановка при добавлении товара в текущий список.
- Долгое нажатие на запись открывает контекстное меню с расширенными действиями: открыть карточку товара, перенести в магазин, переключить срочность, удалить запись.
- Удаление записи и переход к карточке товара убраны из основного UI записи и доступны только через контекстное меню (long press).

### Переключаемые представления текущего списка
- Текущий список поддерживает 4 режима представления: по магазинам, по группам товаров, по получателям, общий без группировки.
- Переключение через горизонтальный ряд FilterChip на экране списка.
- Выбранный режим сохраняется в SharedPreferences (`shopping_list_prefs`) и восстанавливается при повторном открытии.
- Внутри каждого режима работает единая сортировка: срочные некупленные → обычные некупленные → купленные.
- Очистка купленных по группе доступна только в режиме «по магазинам» (canClearBought).
- В режимах «по группам», «по получателям» и «все» в карточке записи показывается название магазина как контекстная информация.
- Модель данных ShopGroup обобщена до EntryGroup, ShoppingListEntryUi расширена полями groupName и recipientName.
- ViewModel использует ClassifierRepository для загрузки групп и получателей.

### Порядок магазинов и сворачивание блоков в текущем списке
- В режиме «по магазинам» пользователь может менять порядок магазинных блоков кнопками вверх/вниз в заголовке каждого магазина.
- Новый порядок сохраняется через поле displayOrder сущности ShopEntity в Room-базе и не сбрасывается при перезапуске.
- Блок «Без магазина» всегда отображается последним и не участвует в перестановке.
- В режиме «по магазинам» каждый магазинный блок можно свернуть/развернуть кликом по заголовку.
- В свёрнутом виде остаётся заголовок магазина с иконкой состояния (стрелка) и счётчиком позиций (активные / всего).
- Состояние сворачивания сохраняется в SharedPreferences (`shopping_list_prefs`) и восстанавливается при повторном открытии.
- Сворачивание и перестановка работают только в режиме «по магазинам» и не затрагивают другие представления.

### Gradle и проектная структура
- `settings.gradle.kts`, `build.gradle.kts` (корневой и app-модуль)
- Gradle wrapper (8.9)
- `local.properties` с путём к Android SDK
- `AndroidManifest.xml` с единственной Activity
- Compile SDK 35, Min SDK 26, Target SDK 35

### Технологический стек
- Kotlin + Jetpack Compose + Material 3
- Room (SQLite) для локального хранения
- Compose Navigation с 3-tab bottom bar
- MVVM (ViewModel + StateFlow)
- Manual DI (AppContainer) без Hilt
- KSP для Room annotation processing

### Модель данных (7 Room-сущностей)
- `ProductEntity` — карточка товара (id, name, productGroupId, recipientId, defaultUnit, defaultQuantity, note, purchaseType, sellerUrl, productUrl, photoUri)
- `ShopEntity` — магазин (id, name, address, note, displayOrder)
- `ShopDepartmentEntity` — отдел магазина (id, shopId, name, displayOrder)
- `ProductShopLinkEntity` — связь товара с магазином (id, productId, shopId, priority, departmentId)
- `ShoppingListEntryEntity` — запись текущего списка (id, productId, quantity, unit, assignedShopId, assignedDepartmentId, isBought, isUrgent, note, createdAt, updatedAt)
- `ProductGroupEntity` — товарная группа (id, name)
- `RecipientEntity` — получатель (id, name)

### DAOs
- 7 DAO-интерфейсов с CRUD-операциями и Flow-запросами
- Поиск по названию товара (LIKE)
- Операции markBought/markNotBought/clearBought/clearBoughtByShop/clearBoughtWithoutShop
- Синхронные запросы для product-shop links и departments (для быстрого переноса между магазинами)

### Репозитории
- `ProductRepository` — товары + связи с магазинами (включая синхронные запросы для переноса)
- `ShopRepository` — магазины + отделы (включая getAllDepartments и getDepartmentsByShopIdSync)
- `ShoppingListRepository` — записи текущего списка (включая clearBoughtByShop/clearBoughtWithoutShop)
- `ClassifierRepository` — товарные группы + получатели

### Экраны (5 обязательных)
1. **Текущий список покупок** — группировка по магазинам (ShopGroup), срочность с визуальным выделением, фильтр «Только срочные», купленные серые и зачёркнутые внизу, очистка купленных по магазину и глобально, бейдж количества справа с быстрым редактированием записи (количество, магазин, заметка), контекстное меню по long press (карточка товара, перенос, срочность, удаление)
2. **База товаров** — список с поиском по названию, кнопка «В список» с диалогом количества и подсказками единиц, FAB для нового товара
3. **Карточка товара** — создание/редактирование всех полей, привязка к магазинам с приоритетом и выбором отдела, подсказки единиц измерения, создание групп/получателей из формы
4. **Список магазинов** — список с порядком отображения, FAB для нового магазина
5. **Карточка магазина** — создание/редактирование, управление отделами (добавление/редактирование/удаление)
