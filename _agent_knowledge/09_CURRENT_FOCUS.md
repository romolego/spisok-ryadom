# 09_CURRENT_FOCUS

**Назначение:** держать текущий практический фокус проекта и помогать агенту не бежать вперёд по несвязанным направлениям.
**Когда читать:** перед определением следующей итерации и перед оценкой приоритета новой задачи.
**Когда обновлять:** если меняется ближайшая цель, следующий шаг, текущий приоритет или набор тем, которые пока нельзя трогать.

## Какая ближайшая цель проекта сейчас

Первая итерация завершена как рабочий базовый продукт. Ближайшая цель — тестирование на устройстве, исправление обнаруженных проблем и поэтапное наращивание функциональности второй очереди.

## Какой следующий шаг должен выполняться

Следующие рабочие шаги:
1. Тестирование на реальном устройстве / эмуляторе — проверка демо-данных, навигации, всех сценариев, включая быстрое редактирование, контекстное меню, перестановку и сворачивание магазинов.
2. Базовая история состояния списка (фиксация факта очистки купленных с датой).
3. UX-улучшения: inline-редактирование дополнительных полей, drag-and-drop сортировка позиций внутри магазина.
4. Подготовка к публикации в RuStore.

## Какой текущий приоритет

Текущий приоритет — стабильная работа первой итерации и постепенный переход к второй очереди улучшений.

## Что сейчас нельзя трогать без отдельного решения

Сейчас нельзя без отдельного решения:
- тащить серверную часть;
- вводить параллельные текущие списки;
- расползаться в сложный UI и вторичную логику;
- тянуть задачи второй очереди раньше стабилизации первой итерации;
- пересматривать базовую концепцию приложения, уже подтверждённую первоисточниками.

## Что уже настроено и не требует повторной проверки без причины

Без новой причины не нужно заново перепроверять:
- наличие git-репозитория;
- наличие helper-скриптов в `tools`;
- наличие двух DOCX-первоисточников в `рабочие материалы`;
- наличие созданной knowledge-инфраструктуры в корне проекта и в `_agent_knowledge`;
- Android-проект компилируется (`./gradlew assembleDebug` проходит);
- 7 Room-сущностей, 7 DAOs, 4 репозитория, 5 экранов реализованы;
- технологический стек: Kotlin + Compose + Room + MVVM + Manual DI;
- демо-данные заполняются при первом запуске;
- текущий список поддерживает 4 режима представления (по магазинам, по группам, по получателям, общий);
- срочность, купленные, быстрый перенос, подсказки единиц — работают;
- быстрое редактирование записи (количество, магазин, заметка) прямо из текущего списка — работает;
- контекстное меню записи (long press): карточка товара, перенос, срочность, удаление — работает;
- перестановка порядка магазинов кнопками вверх/вниз в представлении «по магазинам» — работает;
- сворачивание/разворачивание магазинных блоков с сохранением состояния — работает.
